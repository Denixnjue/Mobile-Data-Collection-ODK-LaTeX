# Digital Data Collection & Technical Reporting Workflow

## Project Overview
This project showcases a complete pipeline for mobile field research. It combines **ODK (Open Data Kit)** for robust data collection and **LaTeX** for professional technical documentation.

## Key Components
*   **XLSForm Design:** A household survey built with logic branching, GPS capture, and data validation constraints.
*   **Image Optimization:** Implementation of the `max-pixels` parameter to ensure field data syncs efficiently in low-bandwidth environments.
*   **Technical Reporting:** A methodology report authored in LaTeX, demonstrating high-standard academic and technical publishing workflows.

## Technical Skills Demonstrated
*   **CAPI/ODK:** Mobile survey architecture and XLSForm standard.
*   **LaTeX:** Typesetting, document structuring, and technical writing.
*   **Field Research Logic:** Designing for data integrity (GPS, constraints, and multimedia verification).

## Files in this Repository
*   `household_survey.xlsx`: The source XLSForm survey design.
*   `Methodology_Report.pdf`: The compiled LaTeX technical report.
*   `Methodology_Report.tex`: The LaTeX source code.
*   `odk_preview.png`: Screenshot of the functional mobile interface.

## How to Use
1.  Upload `household_survey.xlsx` to any ODK-compliant server (e.g., KoboToolbox or ODK Central).
2.  Open the `.tex` file in a LaTeX editor to view the documentation structure.
